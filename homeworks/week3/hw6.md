# 第三週作業    
## 第一題 ：好多星星
這一題我在 CodeWars 寫過類似的，所以寫起來不會太難。不過距離上一次寫程式碼，也是過了三個禮拜，所以有些細節還是會忘記怎麼操作。之前寫題目的時候，都喜歡亂設定變數，搞到自己後來都看不懂在寫什麼，這一次操作就有好好命名，力求自己過了一個月也是一看就知道變數代表的是什麼。

第一題我覺得難的地方不是題目，是要習慣如何用 LIOJ ，這個系統沒辦法跟 Codewars 一樣，給你哪個地方錯誤的提示，搞得我在 submit 的時候都很緊張，總覺得自己跑過一次，用 html debugger 都對，還是會有疏漏的地方！

## 第二題：水仙花數
真的，LIOJ 作業題，只要懂得拆解 Input 的數值，就沒有太難想的地方，只是效率高或低的差別而已。這題只要知道如何算出一個值，後面的迴圈就是看兩者是否相等。

## 第三題：判斷質數
原本以為自己有辦法想出步驟更少的解法，結果後來發現自己想的方向是在一個區間找出質數的做法。

概念是在找到第一個質數之後，只拿這些找到的質數去除以丟進來數字，而不是從頭到尾的數字都測試一遍，才確認其值是質數，不過後來試了一下，還是沒有順利跑出來，就決定先把題目寫完，之後再來思考。

後來思考的解法是，先把 1 拿出來，因為他是特例，每個數除以 1 都會整除，就沒有辨別的作用了。然後從 2 開始看，帶進來的數是否整除，這樣可以第一時間就把很多合數先找出來。

剛開始這樣寫完，還洋洋得意，覺得自己沒有犯下全部找完才確認是質數還合數的爆步驟作法，後來一跑才發現，如果沒有再次確認除數是否等於被除數，就會導致全部的數都變成合數，因為最少有一個數可以整除。

然後是在這題學到了超棒拆解方法 Array.from() 這個函式，可以把字串拆解成陣列，還能進一步對每個陣列的質做變化，不管是要 +2 、 -2，還是要把字串變成數字都沒問題，之後在拆解胡立的資料都是用這樣來拆分使用。

## 第四題：判斷迴文
設計出迴文的字串不難，讓迴圈從尾巴開始跑還第一個就可以，反倒是在設計 Ture 還是 False 的時候，被自己的小聰明騙了！原本以為可以直接 console.log (判斷式)即可，但怎麼測試都錯誤，看 debugger 也覺得自己對啊，產出 ture 跟 false ，後來詢問朋友才知道判斷式跑出來的 ture or false 是布林值，並不是題目要求的字串，而且跑出來的 ture 也是小寫開頭，跟 Output 範例中的 Ture 不同，即便布林值可以轉換成字串，我還是會錯。最後就改成三元條件運算，才順利一行解決！

## 第五題：聯誼順序比大小
跟第四題雷同，覺得自己怎麼那麼聰明，知道直接用 1 跟 -1 來相乘判斷，這樣就不用寫成兩個狀況再做判斷。結果自己把數字帶進去都對，但 LIOJ 就是不給我過，後來看大家的問題才發現， input 可能是很多位的數字，這種情況下，數字會在 JS 裡面被轉換成新的格式，導致程式無法判別哪個大哪個小，後來把字串轉換成 BigInt 的格式後，就順利過關了！
